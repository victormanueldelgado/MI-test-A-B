Test A/B con la función Experimentos de Contenido de Google Analytics.

Un test A/B consiste en probar las respuestas de los usuarios a dos versiones (A y B) de una página. Cuando los usuarios acceden a esa página, a unos se les sirve la versión A y a otros la B sin que ellos sean conscientes de que hay otra versión. Los usuarios interaccionan con la página y el test recoge información de cuántos y qué porcentaje de ellos cumplen el objetivo de esa página para la versión A y para la versión B.

  • Los test A/B pueden incluir más de 2 versiones, podrían incluir n versiones.
  • Cuando un usuario solicita la página, un código javascript proporcionado por Google Analytics decide si a ese usuario le sirve la versión A o la versión B. Pero el usuario no sabe que hay otra versión. Si el usuario vuelve a la página, el código lo detecta y le vuelve a servir la misma versión.
  • El cógigo javascript del experimento hay que incluirlo en todas las versiones de la página al final del HEAD justo antes del final de la etiqueta de cierre </head>.
  • Esas páginas además tienen que tener el código de seguimiento estándar de GA para el sitio web en cuestión (al principio del head justo después de la etiqueta de abertura <head>).
  • El porcentaje de usuarios que realizan la acción que es el objetivo de la página es la tasa de conversión de la página.
  • Cuando ha solicitado la página un número suficiente de usuarios como para que los resultados sean significativos estadísticamente, la versión con mayor tasa de conversión es la más efectiva y la que debería quedarse a formar parte del sitio web. Existen fórmulas para estimar cuántas visitas a la página testeada son necesarias.

El ejercicio:

  • Este ejercicio supone que existe una página dentro de un sitio web que se desea mejorar ( original.htm ).
  • Para ello se crea una versión que se supone que mejora al anterior ( version1.htm ).
  • Los usuarios solictan la página desde un enlace a la misma que se encuentra en la página de inicio del sitio ( inicio.html )
  • El objetivo de la página a mejorar es que el usuario haga click en un enlace tras lo cual se presenta otra página de confirmación ( conversion.htm ). Si el usuario llega a visualizar esta página esto prueba que ha realizado la acción objetivo.

Objetivos:

Los objetivos de una página, aquello que se desea que los usuarios que la visitan realicen, pueden ser muy diversos. Algunos ejemplos:

  • prestar atención al contenido (durante un tiempo mínimo)
  • descargar un documento
  • visualizar un vídeo incluido en la página
  • añadir un producto a un carrito de la compra
  • registrarse (proporcionar el email) para recibir información
  • etc.

GA proporciona 2 mecanismos fáciles para definir objetivos (tiempo de la visita y visita a una página) y uno menos fácil que implica insertar código de seguimiento adicional en los elementos de la página con los que el usuario ha de interaccionar para cumplir el objetivo, para que dicho código envíe datos a GA cada vez que el usuario realiza con ese objeto la acción deseada.
